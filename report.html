1. تطور المشروع والمرجعية

الاتفاقية الأولية (2200 ريال)

موقع قائم على ووردبريس بميزات أساسية:

  • قائمة طعام ثابتة
  • نموذج اتصال بسيط

النطاق الموسع (الحالي)

طلب العميل ميزات متقدمة غير ممكنة في ووردبريس:

  • تخصيص الوجبات فوريًا - أداة بناء السلطة
  • خطط اشتراك متميزة مع تتبع الحميات الغذائية والحساسيات
  • توثيق متعدد المستويات (جوجل/رقم الهاتف)
  • لوحة تحكم إدارية لإدارة المخزون والخطط

التحديات الحرجة التي تم تجاوزها

بدون تصاميم جاهزة

تم بناء جميع الواجهات من الصفر باستخدام Chakra UI + Framer Motion

هجرة قاعدة البيانات

الانتقال من Firebase (NoSQL) إلى Supabase (SQL) بسبب تأخير العميل في تقديم بطاقة ائتمان، مما استلزم إعادة تصميم كاملة

متطلبات متغيرة

أكثر من 5 تعديلات جوهرية خلال التطوير

2. أبرز الإنجازات (تجاوزت النطاق الأصلي)

✅ تجربة المستخدم (وظيفية بالكامل)

الصفحة الرئيسية التفاعلية

  • سلة صور متحركة + عارض وجبات (Framer Motion)
  • تصميم متجاوب (جوال/كمبيوتر)

نظام قائمة ذكي

  • أداة بناء السلطة: حساب الأسعار فوريًا للمكونات الأساسية والإضافات المميزة
  • تصفح حسب التصنيف (سلطات/عصائر/فواكه)

محرك الاشتراكات المميزة

  • فلترات للحميات (كيتو/نباتي/خالي من الجلوتين)
  • متتبع الحساسيات + تقويم جدولة التوصيل
  • توثيق عبر جوجل (توثيق رقم الهاتف جاهز للتكامل)

جمع فعال لبيانات العميل

  • تقرير كامل للمعلومات الأساسية لكل عميل
  • متتبع الحساسيات + تقويم جدولة التوصيل لكل عميل
  • إدخال بيانات الموقع وإضافته للحساب الشخصي باستخدام Google Maps

✅ النظام الإداري (فعّال)

لوحة تحكم مدعومة بـ Supabase

  • عمليات CRUD كاملة للوجبات والمخزون والاشتراكات
  • استيراد/تصدير بيانات جماعي لجميع الكيانات
  • جدولة التوصيل مع تعيين جغرافي

إدارة الخطط

  • تسعير مخصص وتخصيص وجبات للاشتراكات

✅ الهيكل التقني

الواجهة الأمامية: React + Chakra UI (تصاميم مخصصة) + Framer Motion

الواجهة الخلفية: Supabase (قاعدة بيانات SQL + توثيق)

إدارة الحالة: Redux Toolkit + Context API

3. حالة التقدم

الميزة الحالة
أداة بناء السلطة ✅ مكتملة (حساب أسعار فوري)
الخطط المميزة ✅ مكتملة (الحساسيات/حاسبة الوزن المثالي* قيد التطوير)
العمليات الإدارية (CRUD) ✅ مكتملة (أدوات بيانات جماعية (رفع او تنزيل في جداول اكسيل) )
التوثيق (OAuth) ✅ جوجل مكتمل، رقم الهاتف (جاهز للتكامل)
بوابة عمال التوصيل 🔄 جاهزة للتكامل
تحليلات المبيعات 📋 قابلة للنقاش (المرحلة الثانية)
ملاحظة: حاسبة الوزن المثالي (BMI) قيد التطوير (دمج الطول/الوزن/مستوى النشاط)

5. الخطوات القادمة

المرحلة الأولى

  • إكمال حاسبة الوزن المثالي + بوابة إدارة الخطط
  • هجرة الميزات المتبقية (توثيق رقم الهاتف، بوابة التوصيل)

  • التكامل مع نظام Foodics للإدارة للتحديثات الفورية
